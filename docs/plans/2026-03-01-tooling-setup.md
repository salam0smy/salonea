# Tooling Setup Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Install and configure all modules, establish the feature-first directory structure, and set up the foundational patterns (layouts, middleware, repositories) that every subsequent feature will build on.

**Architecture:** Nuxt 4 with domain-organized directories (`components/booking/`, `components/admin/`, etc.), Supabase for data + auth, a repository pattern in `server/utils/repositories/` as the migration seam to Drizzle later, and Arabic as the default RTL locale.

**Tech Stack:** Nuxt 4, @nuxt/ui v3 (Tailwind v4), @nuxtjs/supabase, @nuxtjs/i18n, @pinia/nuxt, @nuxt/eslint, Vitest + @nuxt/test-utils

**Design reference:** `docs/plans/2026-03-01-tooling-setup-design.md`

**Step 3: Implement the pure utility function**

```ts
// server/utils/resolveTenant.ts

const SYSTEM_PATHS = ['/admin', '/api', '/login', '/confirm', '/_nuxt']

export function resolveTenantSlugFromPath(path: string): string | null {
  const segment = path.split('/')[1]

  if (!segment) return null
  if (SYSTEM_PATHS.some(p => path.startsWith(p))) return null

  return segment
}
```

**Step 4: Run tests — verify they pass**


Expected: 5 passed.

**Step 5: Create the server middleware that uses the utility**

```ts
// server/middleware/tenant.ts
export default defineEventHandler(async (event) => {
  const slug = resolveTenantSlugFromPath(event.path)

  if (!slug) return

  // TODO: look up tenantId from slug in the database
```

## Task 11: Set up repository base

**Files:**
- Create: `server/utils/supabase.ts`
- Create: `server/utils/repositories/tenants.ts`

**Step 1: Create server-side Supabase client utility**

```ts
// server/utils/supabase.ts
import { serverSupabaseClient } from '#supabase/server'
import type { H3Event } from 'h3'

export { serverSupabaseClient as useSupabaseServerClient }

// Convenience: typed wrapper used by all repositories
export async function getServerClient(event: H3Event) {
  return serverSupabaseClient(event)
}
```

**Step 2: Create tenants repository**

```ts
// server/utils/repositories/tenants.ts
import type { H3Event } from 'h3'

export interface Tenant {
  id: string
  slug: string
  name: string
  logo_url: string | null
}

export async function getTenantBySlug(event: H3Event, slug: string): Promise<Tenant | null> {
  const client = await getServerClient(event)
  const { data, error } = await client
    .from('tenants')
    .select('id, slug, name, logo_url')
    .eq('slug', slug)
    .single()

  if (error || !data) return null
  return data
}
```

> This is the pattern all repositories follow: accept `event: H3Event` as first argument to get the Supabase client, then `tenantId` or domain-specific params. When migrating to Drizzle, replace the Supabase client call with a Drizzle query — the function signature stays the same.
